7. Requisitos Funcionais (RFs):
RF1 - O usuário deve passar por um processo de login e autenticação do usuário antes de ter acesso à aplicação.
RF2 - O requisitante deve conseguir entrar em contato com o atendente para dar início à própria demanda.
RF3 - O usuário atendente das tarefas deve conseguir acessar uma lista com as prórpias tarefas, organizadas por título, status e prioridade.
RF4 - O usuário atendente das tarefas deve conseguir acessar o detalhamento de cada uma de suas tarefas contendo título, status, prioridade e situação (descrição)
RF5 - O usuário atendente também deve conseguir criar suas próprias tarefas.
RF6 - O usuário atendente deve conseguir finalizar suas próprias tarefas.
RF7 - O usuário atendente deve conseguir abrir uma requisição de deleção das suas próprias tarefas.
RF8 - O usuário atendente deve conseguir definir qual o nível de prioridade das suas próprias tarefas.
Requisitos Não Funcionais (RNFs):
RNF1 - O sistema deve suportar um limite superior a 500 tarefas.
RNF2 - Deve ser acessível via web.
RNF3 - O cliente (FRONTEND) do sistema deve ser construído usando o framework React-JS.
RNF4 - A parte de gerenciamento do servidor (BACKEND) deve ser uma API REST .NET.
RNF5 - O sistema deve ser integrado com banco de dados postgres
RNF6 - As senhas devem ser armazenadas criptografadas.
